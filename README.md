# RNN

**Краткое резюме**
-
- В рамках учебно-практической работы реализовал и интегрировал все ключевые компоненты простого генеративного NLP‑проекта: подготовка корпуса и токенизатор, архитектура RNN/LSTM, обучение/валидация и инференс, в частности было выполнено:
	- инженерия данных и препроцессинг текстовых корпусов;
	- проектирование нейросетевых архитектур;
	- практическая работа с PyTorch: `Dataset`/`DataLoader`, `state_dict`‑based checkpoints, организация обучения и валидации;
	- генерация текста с семплингом.

**Краткое содержание**
- **Скрипты и модули**: `dataset.py`, `model.py`, `train.py` — реализация датасета, модели и логики обучения/валидации/визуализации.
- **Jupyter Notebook**: `genRNN.ipynb` — ноутбук с экспериментами и демонстрациями запуска обучения/инференса.
- **Токенизаторы**: файлы `*.model`, `*.vocab` — предобученные SentencePiece модели и словари.
- **Контрольные точки**: файлы `*_checkpoint.pt` — чекпойнты весов моделей и/или токенайзеров.
- **Датасет**: `jokes.txt` — текстовый файл с корпусом, используемым для обучения/валидации.


Замечания по реализации
-
- `TextDataset` автоматически обучает/загружает SentencePiece модель при первом запуске и разделяет данные на тренировочную и валидационную выборки.
- В `model.py` реализован генератор, который аккумулирует скрытое состояние RNN и последовательно семплирует токены до достижения EOS или максимальной длины.
- В `train.py` реализованы функции для подсчёта и отображения loss и perplexity; для визуализации используется matplotlib + seaborn.

**Стек**
-
- Язык программирования: Python 3.8+
- Фреймворк обучения: PyTorch
- Токенизация: SentencePiece (BPE / Unigram)
- Визуализация: Matplotlib, Seaborn
- Утилиты: tqdm, IPython
- Формат чекпойнтов: сериализация словаря через `torch.save` с ключами `epoch`, `model_state_dict`, `optimizer_state_dict`.
